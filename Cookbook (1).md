## 🚀 Общий шаблон использования

Все примеры ниже используют единую базовую структуру. Для запуска любого use case:
1. Скопируйте шаблон ниже
2. Замените блок `UploadOptions` на нужный вариант из раздела [Use Cases](#-use-cases)
3. Подставьте свои `admin_token`, `email` и путь к файлу

```python
from lexify_api import LexifyAPI, UploadOptions
from pathlib import Path

# === НАСТРОЙКИ ===
ADMIN_TOKEN = "your_lexify_token"  # Запросите на lexiqo_app@mail.ru
USER_EMAIL = "user@example.com"     # Ваш email
INPUT_FILE = "document.docx"        # Путь к проверяемому файлу
# =================

# Инициализация клиента API
api = LexifyAPI(admin_token=ADMIN_TOKEN)

# Получение токена пользователя
user_token = api.get_user_token(USER_EMAIL)

# === ПАРАМЕТРЫ ПРОВЕРКИ (меняются в каждом use case) ===
options = UploadOptions(
    style="По умолчанию",
    format="ГОСТ",
    dictionary="По умолчанию",
    skip_pages=1,
    ai_detector="Включить"
)
# =======================================================

# Загрузка файла и ожидание обработки
upload_data = api.upload_file(user_token, INPUT_FILE, options)
url = api.wait_until_ready(upload_data['file_token'])

# Скачивание результата
output_name = f"Lexify_{Path(INPUT_FILE).name}"
result_path = api.download_file(url, output_name)

# Проверка результата
if result_path and result_path.exists():
    print(f"✅ SUCCESS: Результат сохранен в {result_path}")
else:
    print("❌ FAILED: Не удалось получить результат")
```

> 💡 **Совет:** Сохраните этот шаблон как `check_template.py` и меняйте только блок `UploadOptions` под нужную задачу.

### 16.  Пропуск первых страниц

Пропуск титульного листа, оглавления и других служебных страниц.

```python
options = UploadOptions(
    style="По умолчанию",
    format="ГОСТ",
    dictionary="По умолчанию",
    skip_pages=3,             # ← Пропустить 3 первые страницы
    ai_detector="Включить"
)
```

> 💡 **Рекомендации по skip_pages:**
> - `1` — только титульный лист
> - `2` — титульный + оглавление
> - `3+` — для документов с введением, рефератом и т.п.

##  Справочник параметров

### `style` — Стиль оформления документа
| Значение | Описание |
|----------|----------|
| `"По умолчанию"` | Стандартные правила оформления |
| `"Практика ИВТ"` | Специфичные правила для отчетов ИВТ |
| `"Отключить"` | Не применять проверку стиля |

### `format` — Формат библиографии
| Значение | Описание |
|----------|----------|
| `"ГОСТ"` | ГОСТ Р 7.0.100-2018 (Россия) |
| `"Elsevier"` | Стиль журналов Elsevier |
| `"Harvard"` | Гарвардский стиль (автор-дата) |
| `"IEEE"` | Стиль Института инженеров электротехники и электроники |
| `"MDPI"` | Стиль издательства MDPI |
| `"Springer"` | Стиль издательства Springer |
| `"Отключить"` | Не проверять библиографию |

### `dictionary` — Словарь для проверки терминов
| Значение | Описание |
|----------|----------|
| `"По умолчанию"` | Базовый словарь русского языка |
| `"Практика ИВТ"` | Словарь с терминами кафедры ИВТ |
| `"Отключить"` | Не использовать расширенный словарь |

### `ai_detector` — Детектор ИИ-текста
| Значение | Описание |
|----------|----------|
| `"Включить"` | Проверять текст на генерацию нейросетями |
| `"Отключить"` | Не выполнять детекцию ИИ |

### `skip_pages` — Пропускаемые страницы
| Тип | Диапазон | Описание |
|-----|----------|----------|
| `int` | `0` – `10` | Количество страниц с начала документа, которые нужно игнорировать при проверке |
